- Extracted `ICalendarSource` interface (just `GetBusySlotsAsync`) so multiple calendar sources can be registered.

- `GraphService` implements both `IGraphService` and `ICalendarSource`.
- New `GoogleCalendarService` implements `ICalendarSource` using OAuth 2.0 refresh token. Gracefully no-ops if credentials absent.
- Both `GetAvailableSlots` and `CreateBooking` now aggregate busy slots from all `ICalendarSource` registrations (Outlook + Google merged).
- `Program.cs` updated to register `GraphService` under both interfaces and `GoogleCalendarService` as a second `ICalendarSource`.

- Added one-time token helper tool at `tools/get-google-token/` — run once locally to generate a refresh token.
- All tests updated and passing: 13 passed, 0 failed.

**Important note on Google account type**
- Domain-wide delegation (service account approach) requires Google Workspace. Mike does NOT have Workspace.
- The correct approach for a personal Google account is OAuth 2.0 with a stored refresh token — implemented above.
- The refresh token does not expire unless revoked or unused for 6+ months.

### What Mike needs to do to activate Google Calendar checking
1. Go to [console.cloud.google.com](https://console.cloud.google.com), create a project, enable Calendar API.
2. Create OAuth 2.0 credentials (type: Desktop app). Note Client ID and Secret.
3. Run `cd tools/get-google-token && dotnet run` — browser opens, approve access, copy the refresh token printed.
4. Add to `api/local.settings.json` (Values section):
   - `GoogleClientId`
   - `GoogleClientSecret`
   - `GoogleRefreshToken`
   - `GoogleCalendarId` (use `"primary"` for your default Google calendar)
5. For production: add same values as Azure Function App application settings.
